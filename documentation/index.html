Documentação - Creator QRCode

O Creator QRCode é uma ferramenta projetada para gerar QR codes em lote a partir de diversas entradas de texto. Ele pode ler links ou qualquer texto simples da linha de comando, de um arquivo, de uma planilha CSV ou até mesmo de dados "piped", transformando cada entrada em uma imagem de QR code escaneável e salvando-a como um arquivo PNG em uma pasta organizada.

🚀 Características Principais

  • Múltiplas fontes de entrada: CLI, arquivos de texto, CSV e dados piped
  • Processamento em lote: Geração simultânea de centenas de QR codes
  • Configurações personalizáveis: Tamanho, bordas, correção de erro
  • Organização automática: Estrutura de pastas e nomenclatura inteligente

Capítulo 1: Command Line Interface (CLI)

Bem-vindo ao tutorial do projeto Creator_QRCode! Neste primeiro capítulo, vamos desvendar o mistério por trás da Interface de Linha de Comando (CLI). Pense na CLI como o "painel de controle" do seu programa, a forma como você, o usuário, dá instruções diretas para o nosso gerador de QR code sobre o que fazer.

O que a CLI resolve?

Imagine que você tem um robô super inteligente que cria QR codes. Como você diria a ele: 1. "Crie um QR code para meu site: https://meuprojetomaneiro.com"; 2. "Salve-o em uma pasta especial chamada meus_qrs"; 3. "Faça o QR code bem grande, escale-o para 10!".

Sem uma CLI, você teria que mudar linhas de código dentro do programa toda vez que quisesse criar um QR code diferente. Isso seria tedioso e confuso! A CLI resolve isso permitindo que você digite instruções simples diretamente no terminal do seu computador, como se estivesse conversando com o robô.

Seu Primeiro Comando: Gerando um QR Code

Vamos começar com um exemplo prático. Nosso programa se chama qr_batch_generator.py. Para usá-lo, você abre um terminal (no Windows, pode ser o PowerShell ou Prompt de Comando; no macOS/Linux, geralmente é chamado de Terminal ou Konsole). Uma vez aberto, navegue até a pasta onde o qr_batch_generator.py está salvo e digite seu primeiro comando:

python qr_batch_generator.py "https://www.exemplo.com"

O que acontece quando você pressiona Enter?

  • python: Diz ao seu computador para executar um programa Python.
  • qr_batch_generator.py: O nome do nosso programa.
  • "https://www.exemplo.com": O **input** que você está dando ao programa – o link que você quer transformar em um QR code.

Depois de executar isso, você encontrará uma nova pasta chamada qrcodes (por padrão) no mesmo diretório, contendo uma nova e brilhante imagem de QR code para https://www.exemplo.com!

Entendendo os "Botões" da CLI

A CLI não serve apenas para fornecer entrada. Ela também permite que você ajuste configurações. Essas configurações são frequentemente chamadas de **opções** ou **flags**. Elas geralmente começam com um traço duplo (--) ou um traço simples (-) seguido de uma letra curta.

Vamos ver alguns "botões" importantes para o Creator_QRCode:

BotãoO que ele fazExemplo de uso
urlsOs links que você quer transformar em QR codes.python programa.py "link1"
--outdir (-o)Onde salvar as imagens dos QR codes.--outdir meus_qrs
--scaleQuão grande a imagem do QR code deve ser.--scale 10
--borderQuanto espaço em branco colocar ao redor do QR code.--border 6
--input (-i)Ler links de um arquivo de texto ou CSV.--input links.txt

Exemplo: Digamos que você queira gerar um QR code para https://docs.python.org que seja muito grande e salvo em uma pasta chamada qrs_python.

python qr_batch_generator.py "https://docs.python.org" --outdir qrs_python --scale 15

Aqui:

  • "https://docs.python.org" é o link (**argumento**).
  • --outdir qrs_python usa a **opção** --outdir para especificar a pasta de saída.
  • --scale 15 usa a **opção** --scale para tornar o QR code maior.

A CLI pega todas essas informações e as organiza de forma clara para o programa usar.

Capítulo 2: Orquestrador de Fluxo de Trabalho da Aplicação

No Capítulo 1, você aprendeu a ser o chefe, dizendo ao nosso programa o que fazer usando comandos. Mas o que acontece *depois* que você aperta um botão nesse controle remoto? É aí que entra o **Orquestrador de Fluxo de Trabalho**!

O que o Orquestrador resolve?

Imagine que você está construindo uma nave de LEGO. Você tem as instruções (da CLI), um monte de peças (suas URLs) e ferramentas para juntá-las (o gerador de QR code). Quem garante que você vai ler as instruções primeiro, encontrar todas as peças certas, construir cada parte corretamente e juntar tudo no final?

Esse "quem" é o **Orquestrador de Fluxo de Trabalho**. Ele é o "gerente de projeto" ou o "cérebro" do nosso programa. Ele pega suas instruções da CLI e garante que cada passo da geração do QR code aconteça na ordem correta, do começo ao fim. Sem ele, o programa seria como um trabalhador que recebe as instruções mas não sabe o que fazer primeiro!

O Trabalho do Orquestrador: Um Plano de Alto Nível

O trabalho do Orquestrador pode ser dividido nestes passos principais:

  1. Ouvir as Instruções: Ele ouve atentamente o que você digitou no terminal, entendendo os links e as configurações escolhidas.
  2. Coletar Todas as URLs: Ele garante que tem todos os links para converter, chamando a parte do programa de Gerenciamento de Entrada de URLs para fazer isso.
  3. Criar os QR Codes (Um por Um): Para cada link coletado, o Orquestrador pede ao Renderizador de QR Code para desenhar a imagem.
  4. Salvar e Relatar: Ele garante que os arquivos sejam salvos corretamente, trabalhando com o Manipulador de Arquivos de Saída. Por fim, ele relata a você o que aconteceu: quais QR codes foram criados e onde foram salvos.

Um Olhar por Dentro: A Função `main`

Na nossa arquitetura, a função main é o coração do nosso Orquestrador. É lá que todos os passos que discutimos são colocados em prática.

# Parte da função main()
# 1. Ouvir as Instruções (da CLI)
args = parser.parse_args(argv)

# 2. Coletar Todas as URLs
urls = collect_urls(args)

# 3. Preparar a pasta de saída
outdir = Path(args.outdir)
outdir.mkdir(parents=True, exist_ok=True)

# 4. Criar QR Codes (Um por um)
for u in urls:
    try:
        p = generate_qr_png(u, outdir, ...)
        # ... e salvar o arquivo
    except Exception as e:
        print(f"[ERRO] Falha ao gerar QR para '{u}': {e}")

# 5. Relatar o resultado final
if created:
    print(f"[OK] {len(created)} QR Code(s) gerado(s) em: {outdir.resolve()}")
else:
    print("[ERRO] Nenhum arquivo gerado.")

Este código simplificado mostra como o Orquestrador orquestra a chamada de cada componente, garantindo que tudo funcione em sequência e tratando erros no processo.

Capítulo 3: Gerenciamento de Entrada de URLs

O sistema coleta e organiza as URLs ou textos que serão transformados em QR codes. Ele elimina duplicatas, limpa dados e garante que cada link esteja pronto para a próxima etapa.

O Que o Gerenciamento de Entrada de URLs Resolve?

O componente de Gerenciamento de Entrada de URLs é como seu "Preparador de Ingredientes" dedicado. O trabalho dele é ir a diferentes fontes (linha de comando, arquivos, entrada `piped`), juntar todos os links, remover duplicatas e filtrar o que não é necessário, entregando uma lista limpa e pronta para o próximo passo.

Formas de fornecer URLs:

  • Diretamente na linha de comando
  • Arquivo de texto (.txt)
  • Coluna específica de CSV
  • Entrada "piped" (stdin)

Exemplo de arquivo CSV:

ProductID,Name,Price,URL
101,Laptop,1200,https://shop.com/laptop
102,Mouse,25,https://shop.com/mouse
103,Keyboard,75,https://shop.com/keyboard

Função central: collect_urls

Você pode também gerar QR codes para texto simples usando a opção --allow-plain.

python qr_batch_generator.py "Olá Mundo" --allow-plain

Capítulo 4: Renderizador de QR Code

O Renderizador é o "artista" do projeto. Ele transforma URLs ou textos em QR codes usando a biblioteca segno, aplicando tamanho, borda e correção de erros.

O Que o Renderizador de QR Code Resolve?

O Renderizador é o "artista" ou "pintor" do nosso projeto. Ele pega os dados brutos (como "https://meusite.com") e os transforma na imagem quadrada e pixelada familiar que chamamos de QR code, usando a biblioteca `segno`.

Opções de estilo:

OpçãoFunçãoExemplo
--scaleTamanho do QR code10
--borderEspaço em branco ao redor6
--eccCorreção de erro (l,m,q,h)h
python qr_batch_generator.py "https://example.com/art" --scale 15 --border 8 --ecc h
O Renderizador pega seu dado e cria a arte final em QR code, garantindo que seja escaneável e durável.

Função central: generate_qr_png(data, outdir, prefix, scale, border, ecc)

Capítulo 5: Manipulador de Arquivos de Saída

O Manipulador de arquivos é o "arquivista" que organiza todos os QR codes gerados, garantindo que sejam salvos em pastas corretas com nomes seguros e opcionais prefixos.

O Que o Manipulador de Arquivos de Saída Resolve?

O Manipulador de Arquivos de Saída é como um "arquivista" para seus QR codes gerados. O trabalho dele é garantir que haja uma pasta de destino, dar a cada arquivo um nome seguro e descritivo e, por fim, salvá-lo corretamente.

Opções de saída:

OpçãoFunçãoExemplo
--outdir (-o)Pasta onde os QR codes serão salvosmy_qrs
--prefixPrefixo nos nomes dos arquivoslink_
python qr_batch_generator.py "https://example.com/project" --outdir final_qrs --prefix proj_
O Handler garante que cada arquivo seja único, organizado e fácil de localizar, evitando conflitos e problemas de salvamento.
